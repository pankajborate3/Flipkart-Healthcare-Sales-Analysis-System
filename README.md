1. Introduction

The Flipkart Healthcare Sales Analysis System is a database-driven project designed to manage and analyze pharmacy sales data efficiently. The system simulates healthcare product transactions from an e-commerce platform similar to:

Flipkart

In the modern healthcare retail sector, managing medicine inventory, tracking expiry dates, calculating revenue, and analyzing sales trends are critical for business success. Manual tracking methods are inefficient and prone to human errors.

This project focuses on developing a structured SQL-based system that stores sales transactions, medicine details, customer information, and inventory records. The system generates analytical insights such as total revenue, category performance, top-selling medicines, and expiry alerts.

The primary objective of this project is to create an organized, secure, and efficient healthcare sales management system using SQL and data analytics tools.

---

2. Objectives of the Project

The main objectives of the Flipkart Healthcare Sales Analysis System are:

• To design and manage a relational database using MySQL • To store medicine sales transaction data securely • To perform CRUD operations (Create, Read, Update, Delete) • To calculate total revenue and profit • To analyze category-wise sales performance • To identify top-selling medicines • To monitor medicine expiry dates

• To detect low-stock situations • To generate analytical reports using Power BI • To support data-driven decision-making

The system aims to improve inventory management and financial performance analysis.

4. Dataset Description

The dataset used in this project contains structured healthcare sales data. It is designed to store transaction-level details of medicines sold through an online platform.

The database consists of the following tables:

• Customer Table – Stores customer ID, name, contact details • Medicine Table – Stores medicine name, category, price, expiry date • Sales Table – Stores order ID, customer ID, medicine ID, quantity, order date • Category Table – Stores medicine category information • Inventory Table – Tracks stock quantity and reorder level • Admin Table – Stores administrator login credentials

Each record includes medicine name, category, quantity sold, price, expiry date, order date, total amount, and stock level.

The dataset is normalized to:

• Avoid data redundancy • Maintain referential integrity • Ensure efficient data retrieval • Improve data consistency

5. Dashboard Overview

The Power BI dashboard developed for this project provides a clear and interactive overview of healthcare sales performance.

The dashboard includes:

• Total Revenue • Total Orders • Total Quantity Sold • Category-wise Revenue Distribution • Top-Selling Medicines • Monthly Sales Trend Analysis • Expiring Soon Medicines • Low Stock Alerts • Customer Purchase Analysis • Revenue Growth Percentage

Interactive filters allow users to analyze sales based on:

• Medicine category • Time period (Monthly / Yearly) • Customer • Product type

The dashboard transforms raw transaction data into meaningful visual insights that assist business managers in making informed decisions.

The healthcare sales analysis provides several valuable business insights:
